Repository Configuration

User/Organization Site vs Project Site

GitHub Pages supports two types of sites:

User/Organization Site

  • Repository must be named username.github.io
  • Content is served from the main branch
  • URL: https://username.github.io

Project Site

  • Repository can have any name
  • Content is served from docs folder or gh-pages branch
  • URL: https://username.github.io/repository-name

Creating a New Repository

If you need to create a new repository:

  1. Go to your GitHub dashboard
  2. Click the "+" icon and select "New repository"
  3. For user sites: Name it username.github.io (exact match)
  4. For project sites: Choose any name
  5. Select "Public" (private repos have limited Pages support)
  6. Initialize with a README

Uploading Files

To upload your index.html file:

  1. Go to your repository on GitHub
  2. Click "Add file" â†’ "Upload files"
  3. Drag and drop your index.html file
  4. Add a commit message
  5. Click "Commit changes"

GitHub Pages Settings

Enabling GitHub Pages

To enable GitHub Pages for your repository:

  1. Navigate to your repository on GitHub
  2. Click on the "Settings" tab
  3. In the left sidebar, click "Pages"
  4. Under "Source", select your deployment source:
    • For user sites: "main" branch
    • For project sites: "main" branch or "gh-pages" branch
  5. Click "Save"

Custom Domain (Optional)

If you're using a custom domain:

  1. In the GitHub Pages settings, enter your custom domain
  2. Update your DNS records to point to GitHub Pages
  3. Wait for DNS propagation (can take up to 24 hours)
  4. Enforce HTTPS for secure connections

Checking Deployment Status

To check if your site is building correctly:

  1. Go to your repository â†’ Actions tab
  2. Look for "pages build and deployment" workflows
  3. Check if the build was successful or if there are errors
  4. If there are errors, check the build logs for details

Quick Fix Checklist

  • Repository named correctly (username.github.io for user sites)
  • GitHub Pages enabled in repository settings
  • Source branch set to "main" (or "gh-pages")
  • index.html file in the root directory
  • Waited at least 5 minutes after pushing changes
  • Cleared browser cache or tried incognito mode

Additional Tips

  1. If you're using a custom domain, make sure it's properly configured in your DNS settings
  2. Check for typos in your repository name or GitHub Pages settings
  3. Try accessing your site from a different browser or device
  4. If you recently changed your repository name, wait for DNS propagation
  5. For project sites, remember that your URL will include the repository name
