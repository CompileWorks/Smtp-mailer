Detailed Steps

Step 1: Create a GitHub Account (if needed)

Go to github.com and sign up for a free account.

Step 2: Create a New Repository

Click the "+" icon in the top right and select "New repository".

Name it "smtp-mailer" (or any name you prefer).

Make it public and initialize with a README.

Step 3: Upload the HTML File

Click "Add file" → "Upload files".

Drag and drop the HTML file or choose it from your computer.

Commit the changes to the main branch.

Step 4: Enable GitHub Pages

Go to your repository's "Settings" tab.

Select "Pages" from the left sidebar.

Under "Source", select "main" branch and click "Save".

Your site will be published at: https://[username].github.io/[repository-name]

Step 5: Set Up EmailJS

Sign up for a free account at emailjs.com.

Add and verify your email service (Gmail, Outlook, etc.).

Get your Public Key from the Account section.

Step 6: Configure the Mailer

Open your published GitHub Pages site.

Enter your EmailJS Public Key when prompted.

Configure your SMTP settings through the EmailJS interface.

HTML Code Example

Here's the basic structure of the HTML file you need to upload to GitHub:

// Basic HTML structure for your SMTP mailer <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>SMTP Mailer</title> <script src="https://cdn.jsdelivr.net/npm/@emailjs/browser@3/dist/email.min.js"></script> <style> /* Your CSS styles here */ </style> </head> <body> <div class="container"> <h1>SMTP Mailer</h1> <!-- Your form elements here --> </div> <script> // Initialize EmailJS with your public key emailjs.init("YOUR_PUBLIC_KEY"); // Your JavaScript code here </script> </body> </html>

Remember to replace "YOUR_PUBLIC_KEY" with your actual EmailJS public key.

Important Notes

  1. GitHub Pages hosts static sites only (HTML, CSS, JavaScript)
  2. You cannot run server-side code on GitHub Pages
  3. EmailJS acts as a proxy to handle SMTP requests
  4. For Gmail, you may need to generate an App Password if you have 2FA enabled
  5. Free EmailJS accounts have limited emails per month
  6. For high-volume sending, consider EmailJS paid plans or other services
