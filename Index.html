GitHub Pages automatically looks for an index.html file as the default page for your site. Using this filename ensures your site loads correctly without needing to specify the HTML file in the URL.

Benefits of Using index.html

1. Default Page Loading

Web servers automatically look for an index.html file when a directory is requested. This means visitors can access your site with a clean URL like https://username.github.io instead of https://username.github.io/filename.html.

2. Professional Appearance

Using index.html gives your site a more professional look with cleaner URLs that are easier to share and remember.

3. GitHub Pages Convention

GitHub Pages is specifically designed to look for an index.html file as the landing page for your site. Using this filename follows platform best practices.

4. Future Flexibility

If you later add more pages to your site, using index.html as your main page follows standard web development conventions that will make your site structure clearer.

File Naming Guidelines

Correct Naming

For GitHub Pages, your main HTML file should be named:

index.html (all lowercase, no spaces)

This is the default file that GitHub Pages will serve when someone visits your root URL.

Incorrect Naming Examples

Avoid these naming patterns:

  • Index.html (should be all lowercase)
  • INDEX.HTML (should be all lowercase)
  • smtp mailer.html (spaces in filename)
  • smtp_mailer.html (not the default page)

Repository Structure

Your GitHub repository should have a structure similar to this:

username.github.io/ (repository name)
└── index.html (your main HTML file)
└── README.md (optional documentation)
└── assets/ (optional folder for images, CSS, JS)
    ├── style.css
    └── script.js

GitHub Pages URL Format

When you use index.html as your filename, your GitHub Pages site will be accessible at:

https://username.github.io

If you use any other filename (e.g., smtp-mailer.html), your URL would be:

https://username.github.io/smtp-mailer.html

Using index.html provides the cleanest, most professional URL for your SMTP mailer.

Important Notes for GitHub Pages

  1. Your repository must be named username.github.io (replace "username" with your GitHub username)
  2. Your index.html file must be in the root of the repository, not in a subfolder
  3. GitHub Pages may take 1-2 minutes to publish your site after you push changes
  4. You can use custom domains, but the index.html filename requirement remains
  5. For project sites (not user sites), the repository can have a different name, but index.html is still the default page
